# Dual-Decoder Network With Local Self-Attention for Reparameterization-Based Full Waveform Inversion

This repository contains the code implementation of **LSADD-FWI**, a reparameterization-based full waveform inversion method with a dual-decoder network and local self-attention.

## 1. Required Dependencies

The implementation is performed in PyTorch on an NVIDIA RTX 4090 GPU. The code requires the following Python packages:

- `torch==2.7.1`
- `numpy==1.26.4`
- `matplotlib`
- `deepwave==0.0.20`

All experiments are implemented in PyTorch, and Deepwave is used for wavefield simulation.

## 2. Preparations Before FWI

Before running the inversion examples, the following files should be prepared:

1. The true velocity model, `v_true`.
2. The observed seismic data, `observed_data`, which are generated by forward modeling using `v_true`.
3. The initial velocity model, `v_init`, which can be a smooth model, a linear model, or a constant velocity model.

## 3. Acquisition Setup

The acquisition geometry and source parameters are defined in `_2_preliminary_setting.py`. An example is shown below:

```python
preliminary = Preliminary(
    dx=20.0,                 # Spatial grid spacing
    ny_data=432,             # Number of horizontal grid points
    nx_data=144,             # Number of vertical grid points
    n_shots=50,              # Number of shot gathers
    n_sources_per_shot=1,    # Number of sources per shot
    d_source=5,              # Source spacing
    first_source=90,         # Position of the first source
    source_depth=2,          # Source depth
    n_receivers_per_shot=424,# Number of receivers per shot
    d_receiver=1,            # Receiver spacing
    first_receiver=4,        # Position of the first receiver
    receiver_depth=2,        # Receiver depth
    freq=10,                 # Peak frequency of the Ricker wavelet
    nt=1000,                 # Number of time samples
    dt=0.006,                # Time sampling interval
)
```

## 4. Network Model Definitions

The network architectures are defined in `_3_model.py`.

- `FWI()` represents conventional full waveform inversion, in which the velocity model is optimized directly.
- `CNN_FWI()` represents the CNN-based reparameterization method used for comparison.
- `LSADD_FWI()` represents the proposed LSADD-FWI method, namely a reparameterization network that combines dual decoders with local self-attention.

## 5. Optimization Settings

All experiments are optimized using the Adam optimizer with method-specific initial learning rates. A cosine decay learning-rate schedule is used during optimization. A shot-batch strategy is adopted, with a batch size of 2.

For `FWI()`, the `convention_train_one_step` function is used to update the velocity model directly. For `CNN_FWI()` and `LSADD_FWI()`, the `batch_train_one_step` function is used to update the neural network parameters.

## 6. Running the Experiments

The following scripts are provided:

- `FWI.py`: conventional FWI training.
- `CNN_FWI.py`: CNN-based reparameterization FWI.
- `LSADD_FWI.py`: the proposed LSADD-FWI method.

Each script includes four experimental settings:

1. Inversion from a smooth initial model, corresponding to `M_FWI_inversion_smooth()`.
2. Inversion from a linear initial model, corresponding to `M_FWI_inversion_linear()`.
3. Inversion with noisy observed data, corresponding to `M_FWI_noise_inversion()`.
4. Inversion with observed data lacking low-frequency information, corresponding to `filterd_freq_M_FWI_inversion()`.

Here, `M` denotes the corresponding inversion category.

In `LSADD_FWI.py`, there is also `LSADD_FWI_temp_inversion()`, in which the initial velocity model is a constant model.

The experiments can be run with the following commands:

```bash
python FWI.py
python CNN_FWI.py
python LSADD_FWI.py
```

After inversion, the program saves the final velocity model, the loss curve, and the related result figures.

Several inversion results are provided and saved in the `result` folder. These results can be directly visualized using `visualization.py`, but the observed data need to be generated in advance.
